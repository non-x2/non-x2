## T1. Notionに書き込めない

- **症状**：Notionに運用ボードを作ろうとすると「requires approval」で止まり、書き込みが確定できない。
- **原因**：リモート（クラウド）セッションからのNotion書き込みは承認が通らないことがある。
- **解決**：このリポジトリは **GitHub だけで運用**する方針に決定（PR #1 / `docs/作業ログ_2026-06-21.md`）。Notionで見たい場合は、GitHub上のMarkdownを **Notionに手動Import** すればOK。
- **再発防止**：「Notionボードで管理しよう」と新たに提案しない。資料はMarkdownで作り、必要な人が手元でNotion化する前提にする。

## T2. 作業ブランチが毎回mainから分岐してしまう

- **症状**：同じブランチを使い回していると、PRの差分に**過去にマージ済みの変更まで混ざる**／mainと食い違う。
- **原因**：このリポジトリは PR を **squash（1つにまとめて）マージ**する運用。squashすると元ブランチのコミットIDが変わるため、使い回し（rolling）ブランチは毎回 main から枝分かれしてしまう。
- **解決**：**作業（バッチ）ごとに main から新しいブランチを切る**。
  ```bash
  git checkout main
  git pull origin main                       # 最新のmainにする
  git checkout -b claude/＜今回の内容＞ main   # mainから新しい枝を作る
  ```
- **再発防止**：1テーマ＝1ブランチ。マージが終わった枝は再利用しない。ブランチ名は内容が分かる名前にする（例：`claude/ff5-anime-ep17-20`）。

## T3. `git reset --hard` が拒否される

- **症状**：`git reset --hard` を実行しようとすると、環境のセーフガードに**拒否される**。
- **原因**：作業環境が、戻せない破壊的操作を安全のためブロックしている。
- **解決**：そもそも `reset --hard` が要らない進め方にする。**T2のとおり main から新しい枝を切れば**、ブランチ同期のためのresetは不要。
- **再発防止**：「いったん全部巻き戻す」発想に頼らない。やり直したいときは**新しい枝を切り直す**。間違いを取り消したいときは Claude に「直前の操作を確認して。やり直せる？」と聞く。

## T4. PRマージで「Base branch was modified」

- **症状**：PRをマージしようとすると `Base branch was modified` と出て一度失敗する。
- **原因**：マージ直前に main 側が更新され、PRが見ていたbase（土台）と食い違ったため。
- **解決**：**PRの状態を取り直して、もう一度マージする**とたいてい成功する。慌てず再試行でOK。
- **再発防止**：複数PRを続けてマージするときは、1つマージしたら次のPRを**最新のmainで取り直して**から進める。

## T5. Gammaのスライドが思った形にならない

- **症状**：Gammaのスライドが**10枚で打ち切り**になる／**9:16の縦型**にならず横長(fluid)になる。
- **原因**：Gammaの presentation 形式の制約。
- **解決**：
  - 枚数は **`numCards` を10以下**にする（10枚を超える構成は分割する）。
  - 縦型にしたいときは **`imageOptions.style` に "vertical 9:16"** を入れる（カード自体は fluid になる前提で、画像を縦構図に寄せる）。
- **再発防止**：縦型ショート用の絵コンテは最初から「10枚以内＋縦構図プロンプト」で設計する。既存Gammaの管理IDは `ff5-anime-HANDOFF.md` §4 に控えがある。

## T6. YouTubeの実動画を分析できない

- **症状**：実チャンネルの動画を見て尺・テロップ・サムネを分析しようとすると、ページにアクセスできない。
- **原因**：リモートセッションは**ネットワーク許可リスト**で外部アクセスが制限されており、`youtube.com` 系が未許可だと開けない。
- **解決**：環境作成時の**ネットワーク許可リストに `youtube.com` 系ドメインを追加**してもらう（要ユーザー操作）。
- **再発防止**：実動画分析が必要なタスクは、**着手前に許可リストを確認**。許可が取れない間は、台本・サムネ等の「型づくり」など、ネット参照が不要な作業を先に進める。

## T7. `gh` コマンドが使えない環境がある

- **症状**：`gh pr create` などの `gh`（GitHub CLI）コマンドが実行できない。
- **原因**：セッションの実行環境によっては `gh` / `hub` CLI や直接のGitHub APIが**使えない**。`ff5-anime-HANDOFF.md` には「ghが確実」とあるが、それは別環境での話。
- **解決**：GitHub連携（PR作成・コメント・CI確認など）は、その環境で提供される **GitHub連携ツール（MCPツール）**を使う。Claudeへは「**PRをドラフトで作って**」のように頼めばよく、ツールの違いはClaude側で吸収する。
- **再発防止**：手順書のコマンド例（`gh ...`）は**あくまで一例**と捉える。`gh` が無い環境では同等の操作をMCPツールで行う。Git操作の頼み方は [`指示書_Git操作手順書.md`](指示書_Git操作手順書.md) を参照。
