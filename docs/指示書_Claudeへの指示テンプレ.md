## 🧭 まず覚える「良い指示の3要素」

| 要素 | 質問 | 例 |
|------|------|----|
| **背景・目的** | 何のため？ | 「初心者の自分が学ぶため」 |
| **誰向け** | 読む/使うのは誰？ | 「中年家庭層の視聴者」 |
| **完成形** | どんな形で欲しい？ | 「スライド」「表」「チェックリスト」 |

> この3つを最初に書くだけで、成果物の精度が大きく上がります。

## 🚀 すぐ使える指示テンプレ

### 1. 何かを作ってほしい（基本形）
```
○○を作ってください。
・目的：○○のため
・対象：○○向け（初心者／視聴者 など）
・形式：○○（スライド／Markdown／表 など）
・条件：○○は必ず入れて。専門用語はかみ砕いて。
まず方針を相談してから進めてください。
```

### 2. まず案を出してほしい
```
○○について、案を3つ出してください。
それぞれのメリット・デメリットも添えて。選んでから進めます。
```

### 3. 既存のものを直してほしい
```
○○の△△の部分を、□□に直してください。
他の部分はそのままで大丈夫です。
```

### 4. 調べてまとめてほしい
```
○○について調べて、初心者にも分かるように要点をまとめてください。
たとえ話を入れて、最後に「結論」を一行で。
```

### 5. 画像・資料を渡して反映してほしい
```
（画像やファイルを添付して）
この内容を反映して、○○を更新してください。
```

## 🛠 作業（コード・Git）を頼むとき

### コミット〜PRまで
```
変更ができたらコミットして、GitHubにプッシュ、
PRはドラフトで作成してください。
```

### マージしてほしい / してほしくない
```
（OKのとき）PRをマージしてください。
（待ってほしいとき）プッシュまででOK。マージはまだしないで。
```

### 確認しながら進めてほしい
```
大きい変更は、作る前に方針を確認してください。
戻しにくい操作（公開・削除・マージ）の前は必ず聞いてください。
```

## 💡 精度を上げる小ワザ

- **見せる**：文字で説明しづらい時はスクショ・見本を貼る
- **絞る**：「ポイントは3つ」「300字以内」など範囲を指定
- **任せる**：細部は「いい感じで」「おすすめで」でOK
- **段階的に**：一度に全部より、少しずつ直す方が早い
- **聞き返す**：分からない用語は「○○って何？」で即解説

---

## ⚠️ 安全のために伝えること

| 操作 | 伝え方の例 |
|------|-----------|
| 公開・送信 | 「公開していい／まだ下書きで」 |
| 削除・上書き | 「消す前に内容を見せて」 |
| マージ | 「マージしていい／まだ待って」 |
| 外部サービス | 「○○に送っていい？」 |

> Claudeは戻しにくい操作の前に確認を取りますが、**最初に方針を伝えておく**とスムーズです。
